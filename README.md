Reflection Journal

Throughout these projects, I developed a stronger understanding of how to build reliable, secure, & user-focused software through structured testing & thoughtful design.

To ensure that my code is both functional & secure, I rely heavily on unit testing and validation. In Project 1, I created comprehensive test cases to verify that each class behaved as expected under normal & edge conditions. Writing tests alongside development helped me catch defects early & confirm that updates did not break existing functionality. I also implemented input validation & constraints such as limiting field lengths & preventing null values to reduce the risk of invalid data entering the system. This approach reinforces both correctness and basic security by preventing misuse or unintended behavior.

Interpreting user needs begins with carefully analyzing requirements & translating them into clear, testable conditions. I focused on breaking down each requirement into specific behaviors the program should exhibit. For example, when designing the contact service, I ensured that each field aligned with user expectations, such as having unique IDs & properly formatted data. By mapping requirements directly to test cases, I was able to confirm that the implementation consistently met user needs. This process helped me bridge the gap between abstract requirements and concrete functionality.

When approaching software design, I prioritize modularity, clarity, & scalability. I structured my code using separate classes with well-defined responsibilities, which made the system easier to maintain & extend. For instance, isolating the service layer from the data model allowed me to manage contacts more efficiently & test components independently. I also consider how the design might evolve, ensuring that the structure can accommodate future enhancements without requiring major refactoring. Overall, my design approach emphasizes simplicity, organization, & alignment with requirements.

These projects strengthened my ability to combine testing, design, & requirement analysis into a cohesive development process. They demonstrate not only my technical skills but also my ability to think critically about software quality and user needs.
